# 插件开发小管家 / hanako-chajian-xiaoguanjia

> ⚠️ **用之前先知道这些**
>
> 这个 skill 是我在开发 Hana 插件时反复踩坑后整理出来的。它能把开发过程管理起来——写日志、备份、换窗口交接、翻经验。但很多功能需要你主动说句话它才会动，不是装好就自动全帮你搞定的。

## 它的本意

和 Hana 助手一起开发插件时，我最烦三件事：
1. 换了个窗口，上一轮修到哪全忘了，又得从头说一遍
2. 我不会写代码，出问题了只能靠口述和截图告诉助手，全靠助手自己去诊断和调试
3. 一个功能修好了，回头发现另一个功能不知道什么时候坏了

这个 skill 就是对着这三件事来的。它不花哨，但每件都能帮上忙。

## 它能做的六件事

**写日志** — 每个插件项目自动维护一份 `PROJECT_LOG.md`，记录进度、踩坑、经验。不用你自己写，遇到什么问题直接告诉助手，助手会帮你记下来整理好。换窗口时自动更新，新窗口接上就能读

**测回归** — 新窗口启动时自动跑后端工具，看看之前好的功能有没有被改坏。你只需要截图或者描述一下现象，助手自己去验证。

**备份** — 关键节点提醒你备份，防止改不回去

**翻经验** — 卡住了说一句查坑或翻踩坑，告诉助手你遇到什么问题，它会去匹配踩坑记录里有没有类似的

**写调试日志** — 遇到 bug 你不需要懂技术细节，告诉助手哪里不对，助手自己去后端加诊断标记、查日志、定位问题

**找人审** — 做完了可以叫其他助手来多维审查，你不用懂代码，告诉他们帮你看就行

## 经验是怎么积累的

这是我比较在意的一个设计。

这个 skill 不是 AI 自动发现经验的。它的工作方式是：

```
开发中踩坑 → 你让助手记到日志里
     ↓
找到解法 → 记到成功经验里
     ↓
换窗口 → 日志更新，新经验带过去
     ↓
新窗口 → 先读日志 → 上次的经验自然浮现
```

所以它**越用越厚**，但前提是你愿意配合——踩了坑记得说一声"记一下"。

## 什么时候该说什么

| 你想... | 就说... |
|--------|--------|
| 开始做个插件 | 开发 xx 插件 |
| 卡住了想翻经验 | 查坑 / 翻踩坑 |
| 想后台调试 | 去写点日志来定位 / 去查日志看看 |
| 换窗口 | 换窗口 / 新窗口见 |
| 接着昨天搞 | 继续开发 xx |
| 备份一下 | 备份 |
| 找人帮忙看看 | 找人审一下 / 叫 xxx 来看看 |

## 它解决什么痛点

**换窗口后上下文丢失**
→ 项目日志文件持久化记录当前进度

**不会写代码，只能口述和截图**
→ 助手自己去诊断、调试、记录，你只需要说发生了什么

**修 bug 搞坏其他功能而不自知**
→ 新窗口启动时自动跑后端回归测试

**改坏了回不去**
→ 关键节点自动备份，保留多版本快照

**同一个问题反复遇到**
→ 踩坑和成功经验持续累积，越用越厚

**缺少多视角**
→ 做完后可以叫其他助手审查代码

## 和 hana-plugin-creator 的区别

**hana-plugin-creator** 负责：生成代码骨架（manifest、tools、routes 等文件）
**plugin-dev-guide（这个 skill）** 负责：管理开发流程（日志、备份、审查、交接）

两个可以配合用。
